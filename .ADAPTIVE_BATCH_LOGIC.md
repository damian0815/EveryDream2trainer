The main train.py and core/step.py incorporate an *adaptive step* methodology that decouples *forward batch size*, *backward step interval* (sample count), and *optimizer step interval* (sample count).

This functionality is missing from train_sana. We will need to abstract the existing logic in train.py and core/step.py into a set of reusable components that can be cleanly shared with train_sana.py.

Here is a schematic outline of the logic (INCOMPLETE/INACCURATE, just to get the ball rolling):
